= Validating Query Parameters and Headers
ifndef::env-site,env-github[]
include::_attributes.adoc[]
endif::[]

Configure *APIkit Router* to enforce validation on headers and query parameters, restricting them only to the attributes specified within your API specification.

// tag::process[]
. In your Studio, select *APIkit Router*.
+
image::apikit-router-selection.png["APIkit router highlighted"]
. Click *Edit* for your router configuration.
+
image::router-config.png["Edit button highlighted in the APIkit router tab"]
// end::process[]

. Select *Query Parameters Strict Validations* to restrict the usage of query parameters to only the ones defined in your API specification. 
. Select *Headers Strict Validations* to limit the headers to only the ones specified in your API specification:
+
image::enable-verifications.png["Query parameters strict validations and headers strict validations highlighted in the router configuration tab."]

If not, update the APIkit configuration code as following:

[source,xml,linenums]
----
<apikit:config  name="api-config"
                api="api.raml"
                outboundHeadersMapName="outboundHeaders"
                httpStatusVarName="httpStatus"
                queryParamsStrictValidation="true"
                headersStrictValidation="true" />
----

== Disable Validation of Query Parameters and Headers

If your specific scenario and use case can handle it, disable the validation of query parameters and headers to minimize latency.

include::page$validate-4-task.adoc[tag=process]
. Select *Disable Validations*:
+
image::disable-verifications-apikit.png["Disable verifications option highlighted in the router configuration tab"]

If not, update the APIkit configuration code as following:

[source,xml,linenums]
----
<apikit:config  name="api-config"
                api="api.raml"
                outboundHeadersMapName="outboundHeaders"
                httpStatusVarName="httpStatus"
                disableValidations="true" />
----
