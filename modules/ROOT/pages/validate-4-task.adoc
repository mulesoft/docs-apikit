= Validating Query Parameters and Headers
ifndef::env-site,env-github[]
include::_attributes.adoc[]
endif::[]

You can configure the APIkit Router to enforce validation on headers and query parameters, restricting them only to the attributes specified within your API specification.

// tag::process[]
. In your Studio canvas, select *APIkit Router*.
+
image::apikit-router-selection.png["APIkit router highlighted."]
. Select the *Edit* button for your router configuration
+
image::router-config.png["Edit button highlighted next to the plus button, in the APIkit router tab."]
// end::process[]
. Enable *Query Parameters Strict Validations* to restrict the usage of query parameters solely to those that are defined in your API specification. Similarly, enable *Headers Strict Validations* to limit the headers to only those specified in your API specification:
+
image::enable-verifications.png["Query parameters strict validations and headers strict validations highlighted in the router configuration tab."]

Or just update the APIkit configuration code as following:

[source,xml,linenums]
----
<apikit:config  name="api-config"
                api="api.raml"
                outboundHeadersMapName="outboundHeaders"
                httpStatusVarName="httpStatus"
                queryParamsStrictValidation="true"
                headersStrictValidation="true" />
----

== Disable Validation of Query Parameters and Headers

If minimizing latency is of utmost importance for your specific scenario and your use case can tolerate the absence of validations, it is possible to deactivate them.

include::page$validate-4-task.adoc[tag=process]
. Select *Disable Validations*:
+
image::disable-verifications-apikit.png["Disable verifications option highlighted and checked in the router configuration tab."]

Or just update the APIkit configuration code as following:

[source,xml,linenums]
----
<apikit:config  name="api-config"
                api="api.raml"
                outboundHeadersMapName="outboundHeaders"
                httpStatusVarName="httpStatus"
                disableValidations="true" />
----
