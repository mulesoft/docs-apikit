When using a load balancer in front of a RAML or OAS API implementation, you can enable the *Keep RAML/OAS base URI* flag. This ensures that the URL specified in the proxy matches the one in your API.

// TODO Add an image pointing toi the property in the configuration if necessary

To edit the code, set the `keepApiBaseUri` to `true`:

[source,xml,linenums]
----
<apikit:config  name="api-config"
                api="api.raml"
                outboundHeadersMapName="outboundHeaders"
                httpStatusVarName="httpStatus"
                keepApiBaseUri="true" />
----
