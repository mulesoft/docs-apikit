APIkit creates error-handling code using Mule 4 main mechanism. When something goes wrong with a part of the process, you receive a typed error message. This code covers common HTTP status codes, matched to types and error messages listed in this article.

== Error Types

APIkit for SOAP defines two error types:

[%header]
|===
|Error Type |Description

|APIKIT-SOAP:ROUTER
|Routing exception

|APIKIT-SOAP:FAULT
|Fault exception
|===

== Error Handling

APIkit for SOAP scaffolder does not generate error-handling code.
